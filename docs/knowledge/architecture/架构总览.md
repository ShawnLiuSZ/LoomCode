# 架构总览

> 📅 最后更新：2026-07-14

## 分层架构

```
┌─────────────────────────────────────────────────┐
│              表示层 (Presentation)               │
│      CLI 入口 · TUI 界面 · Web Dashboard         │
├─────────────────────────────────────────────────┤
│              应用层 (Application)                │
│    Agent 引擎 · 推理循环 · 多Agent编排 · 会话     │
├─────────────────────────────────────────────────┤
│              领域层 (Domain)                     │
│  Provider适配 · 工具系统 · 控制层 · 记忆系统      │
├─────────────────────────────────────────────────┤
│            基础设施层 (Infrastructure)            │
│   配置加载 · SQLite存储 · MCP通信 · LSP · HTTP    │
└─────────────────────────────────────────────────┘
```

## 核心设计原则

| 原则 | 说明 |
|------|------|
| **Provider 插件化** | Adapter 工厂模式，配置驱动接入新厂商，无需修改代码 |
| **Capability-Driven** | Agent 行为由 Provider 的 `Capabilities` 声明驱动，非 if-else 判断厂商 |
| **Config Over Code** | 模型、工具、插件全部 JSON 声明 |
| **Compose Over Inherit** | 接口组合构建复杂行为，避免深层继承 |
| **Single Binary** | CGO_ENABLED=0，零依赖部署 |

## 模块依赖关系

```
cmd/loomcode/main.go
    │
    ├── internal/config/          ← 配置系统（被所有模块依赖）
    ├── internal/provider/        ← Provider 层（模型接入）
    ├── internal/tool/            ← 工具系统（注册/执行/修复）
    ├── internal/agent/           ← Agent 引擎（依赖 provider + tool）
    ├── internal/control/         ← 控制层（权限/成本/门控）
    ├── internal/session/         ← 会话管理（JSONL 持久化）
    ├── internal/mcp/             ← MCP 插件（外部工具扩展）
    ├── internal/skills/          ← Skills 管理（自动加载）
    ├── internal/lsp/             ← LSP 集成（语言服务器）
    ├── internal/dashboard/       ← Web Dashboard
    └── internal/ui/              ← TUI 界面（依赖上述所有）
```

## 单次任务数据流

```
用户输入 "读取 main.go 并解释"
    │
    ▼
CLI 入口 (cmd/loomcode/main.go)     ← 解析参数、加载配置
    │
    ▼
Config System                       ← 选择 Provider、模型
    │
    ▼
Agent Loop (agent/loop.go)          ← 构建消息上下文
    │
    ▼
Provider.Stream()                   ← SSE 流式推理
    │
    ├─ 有 tool_calls → RepairPipeline → ToolExecutor → 结果追加 → 继续循环
    │
    └─ 无 tool_calls → 就绪检查 → 流式输出最终答案 → 返回
```

## Agent 模式

| 模式 | 权限 | 适用场景 | 触发方式 |
|------|------|----------|----------|
| **Build** | 完整工具权限 | 日常开发（默认） | 默认模式 |
| **Plan** | 只读分析 | 代码探索、方案设计 | `/plan` 命令 |
| **Compose** | 编排模式 | 规格驱动开发 | `/compose` 命令 |
| **Max** | 并行选优 | 高难度任务 | 配置 `experimental.maxMode` |

## 技术栈

| 层次 | 技术 |
|------|------|
| 语言 | Go（CGO_ENABLED=0） |
| 配置 | JSON + JSON Schema |
| TUI | Bubble Tea + Lip Gloss |
| 记忆存储 | SQLite FTS5（modernc.org/sqlite） |
| 插件协议 | MCP（stdio + HTTP） |
| LSP | JSON-RPC 2.0 |
| API 协议 | OpenAI 兼容 |
| 跨平台 | Unix + Windows（build tags 分流） |

## 配置优先级

```
--config <path>                            ← CLI 参数（最高优先级）
    │
<project>/.loomcode/settings.local.json   ← 项目本地覆盖（gitignore）
<project>/.loomcode/settings.json         ← 项目级共享配置
    │
~/.loomcode/{settings.json, models.json}  ← 全局配置（合并加载）
    │
内置默认值                                ← 最低优先级（旧版 TOML 仅迁移输入，已废弃）
```

## 环境变量加载顺序

```
1. ~/.loomcode/.env        ← 全局配置
2. ./.env                  ← 项目配置
3. ./.env.local            ← 本地覆盖（不提交 git）
4. --env-file custom.env   ← CLI 参数（最高优先级）
```
